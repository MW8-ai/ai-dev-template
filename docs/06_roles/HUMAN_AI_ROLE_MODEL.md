# HUMAN_AI_ROLE_MODEL.md

## Purpose

Define ideal human and AI roles for software development.

## Human Roles

### Product Owner

Owns goals, priorities, acceptance criteria, user value.

### Technical Lead

Owns architecture, quality bar, technical tradeoffs.

### Platform Engineer

Owns environments, CI/CD, secrets, deployments, observability.

### Security Engineer

Owns access control, threat modeling, secrets, audits, compliance review.

### Compliance / Governance Lead

Owns policy alignment, evidence, change control, NIST/FIPS posture.

### AI Orchestrator

Assigns work to agents, chooses models, reviews risk, controls context.

## AI Roles

### Planner Agent

Breaks work into small tasks.

### Builder Agent

Implements scoped changes.

### Tester Agent

Runs checks and reproduces issues.

### Reviewer Agent

Checks scope, safety, quality, and maintainability.

### Documenter Agent

Updates README, changelog, testing, roadmap.

### Research Agent

Checks current official docs, pricing, capabilities, constraints.

### Health Reviewer Agent

Occasionally reviews the whole repo for drift, repetition, and bad loops.

## Principle

AI agents can do work.
Humans own judgment, access, and accountability.
