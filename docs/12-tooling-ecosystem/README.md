# Tooling Ecosystem

This section tracks external tools, official repositories, community examples, MCP servers, AI coding assistants, and LLM model options.

The goal is awareness and safe evaluation — not blind adoption.

External repositories are references. They are not trusted dependencies until reviewed.

---

## Core Rule

> Link first. Review second. Adapt third. Import last.

Do not copy code, skills, agents, workflows, prompts, or MCP servers from external sources into this repository until they have been reviewed for license, maintenance, security, and fit.

---

## What This Section Covers

- How to review external repositories before adoption
- How to evaluate Model Context Protocol (MCP) servers
- How to choose tools for a project or team
- How to track Large Language Model (LLM) options without turning the repo into a stale ranking list

## Beginner Summary

There are many good tools and example repositories for AI-assisted development. Some are official, some are community-maintained, and some are experimental.

This section helps you answer:

- What is this tool?
- Who maintains it?
- Is it safe to use?
- What access does it need?
- Does it fit this project?
- Should we link to it, test it, adapt it, or avoid it?

---

## Expert Summary

This section is the intake and governance layer for AI-era developer tooling.

Use it to separate discovery from adoption:

```text
Discover → Review → Evaluate → Pilot → Adopt → Maintain
```

A popular repository is not automatically safe. An official repository is not automatically production-ready. An MCP server with useful access may also create a large permission boundary.
