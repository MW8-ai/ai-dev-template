# Confusion and Troubleshooting Pack

This folder explains the concepts that commonly confuse new users and still matter to senior developers designing team workflows.

## Recommended Reading Order

1. `MERGE_METHODS_EXPLAINED.md`
2. `FETCH_PULL_PUSH_AND_REMOTE_BRANCHES.md`
3. `PULL_REQUEST_SAFETY_AND_REJECTION.md`
4. `AUTO_PR_AND_AI_AGENT_GUARDRAILS.md`
5. `COMMON_PITFALLS_AND_TROUBLESHOOTING.md`
6. `REAL_WORLD_BLUNDERS_AND_PREVENTION.md`

## Audience

This content is written for:

- beginners learning GitHub
- experienced developers standardizing team practices
- AI-assisted developers using tools like Copilot, Claude Code, Cursor, Codex, Codespaces, and automation agents
- managers and technical leads who need explainable guardrails

## Tone

The goal is not "zero to hero."

The goal is practical literacy:

```text
Know what the button does.
Know why the rule exists.
Know what can go wrong.
Know how to recover.
```
