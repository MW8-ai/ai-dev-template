# Contributing

## Working agreement

Small, reviewable changes are preferred. Every change should explain the problem, the approach, the risk, and the validation performed.

## Branch naming

- `feature/<short-name>`
- `fix/<short-name>`
- `docs/<short-name>`
- `security/<short-name>`

## Pull requests

Use the PR template. Include screenshots, logs, or test evidence when relevant.

## AI-assisted work

AI output must be reviewed by a human before merge. Do not paste secrets, private data, regulated data, or credentials into external tools.
