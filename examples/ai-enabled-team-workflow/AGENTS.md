# Agent Rules for This Example

This file defines how AI agents may operate in this example project.

---

## Allowed

Agents may:

- read documentation and project files
- propose implementation plans
- draft code or documentation changes
- create tests
- summarize Pull Requests
- identify risks and assumptions

---

## Not Allowed Without Human Approval

Agents may not:

- merge Pull Requests
- deploy to production
- access real secrets
- change authentication or authorization
- modify infrastructure
- install MCP servers
- add external dependencies
- delete files or data

---

## Required Output

Agent-assisted work must include:

- summary of changes
- files changed
- tests or checks run
- assumptions
- risks
- human approval needed

---

## Review Rule

> Passing automation does not replace human review.
